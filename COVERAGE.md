# Coverage

Status reflects whether the upstream Tailwind utility is implemented in rwind. 📌 marks utilities likely to see the most use. Plan markers — ➕ Likely to be ported · ➖ Unlikely to be ported · 🔁 Revisit if needed.

| Project            | Description                                                                           | Status       | Plan                    | Notes                                                                                               |
| ------------------ | ------------------------------------------------------------------------------------- | ------------ | ----------------------- | --------------------------------------------------------------------------------------------------- |
| 📌 Font family      | `font-sans`, `font-serif`, `font-mono`                                                | ✔️ Ported     |                         | Maps to `FontFace`                                                                                  |
| 📌 Font weight      | `font-thin` → `font-black`                                                            | ✔️ Ported     |                         | Full Tailwind scale                                                                                 |
| Font style         | `italic`, `not-italic`                                                                | ✔️ Ported     |                         |                                                                                                     |
| 📌 Font size        | `text-xs` → `text-9xl`                                                                | ✔️ Ported     |                         |                                                                                                     |
| Line height        | `leading-none` → `leading-loose`, `leading-3` → `leading-10`                          | ✔️ Ported     |                         |                                                                                                     |
| 📌 Text color       | `text-{hue}-{shade}`, `text-white`, `text-black`, `text-transparent`                  | ✔️ Ported     |                         | Reads full palette                                                                                  |
| Text alignment     | `text-left`, `text-center`, `text-right`, `align-top`, `align-middle`, `align-bottom` | ✔️ Ported     |                         |                                                                                                     |
| Text wrap          | `truncate`, `text-wrap`, `text-nowrap`                                                | ✔️ Ported     |                         |                                                                                                     |
| Text decoration    | `underline`, `line-through`                                                           | ✔️ Ported     |                         | Applied via `decorate(text, classes)`, which wraps content in `<u>` / `<s>` tags and toggles `RichText`. Bare classes have no effect — text must flow through `decorate()` |
| Letter spacing     | `tracking-*`                                                                          | ❌ Not ported | ➖ Unlikely to be ported | `TextLabel` exposes no letter-spacing property                                                      |
| Text transform     | `uppercase`, `lowercase`, `capitalize`                                                | ✔️ Ported     |                         | Applied via `decorate(text, classes)`, which transforms the string at runtime. Bare classes have no effect — text must flow through `decorate()` |
| 📌 Border radius    | `rounded`, `rounded-{none,sm,md,lg,xl,2xl,3xl,full}`                                  | ✔️ Ported     |                         | Via `::UICorner`; `rounded-full` uses `UDim.new(0.5, 0)`                                            |
| Per-corner radius  | `rounded-{t,r,b,l,tl,tr,br,bl}-*`                                                     | ❌ Not ported | 🔁 Revisit if needed     | Waiting on `TopLeftRadius` / `TopRightRadius` / `BottomRightRadius` / `BottomLeftRadius` to ship    |
| Border width       | `border`, `border-{0,2,4,8}`                                                          | ✔️ Ported     |                         | Uniform — `UIStroke` has no per-side widths                                                         |
| 📌 Border color     | `border-{hue}-{shade}`, `border-white`, `border-black`                                | ✔️ Ported     |                         | Via `::UIStroke`                                                                                    |
| Per-side borders   | `border-{t,r,b,l,x,y}-*`                                                              | ❌ Not ported | ➖ Unlikely to be ported | `UIStroke` is uniform on all four edges                                                             |
| Border style       | `border-solid`, `border-dashed`, `border-dotted`, `border-double`                     | ❌ Not ported | ➖ Unlikely to be ported | `UIStroke` has no stroke pattern                                                                    |
| Ring width         | `ring`, `ring-{0,1,2,4,8}`                                                            | ✔️ Ported     |                         | Shares `::UIStroke` with `border-*` — last rule wins, can't stack                                   |
| Ring color         | `ring-{hue}-{shade}`, `ring-white`, `ring-black`                                      | ✔️ Ported     |                         | Same `::UIStroke` collision                                                                         |
| Ring offset        | `ring-offset-{0,1,2,4,8}`                                                             | ✔️ Ported     |                         | Maps to `::UIStroke.BorderOffset`; only takes effect alongside `ring-*` (Border mode)               |
| Ring position      | `ring-inset`, `ring-outer`, `ring-center`                                             | ✔️ Ported     |                         | Maps to `::UIStroke.BorderStrokePosition` (Inner/Outer/Center); `outer`/`center` are Roblox-only    |
| Outline            | `outline-*`                                                                           | ❌ Not ported | ➖ Unlikely to be ported | Would be a third utility colliding on `::UIStroke`                                                  |
| Drop shadow        | `shadow`, `shadow-{none,sm,md,lg,xl,2xl}`                                             | ✔️ Ported     |                         | Via `::UIShadow`; rendering is gated behind a Roblox FFlag — invisible until the engine flag flips  |
| Shadow color       | `shadow-{hue}-{shade}`, `shadow-white`, `shadow-black`                                | ✔️ Ported     |                         | Same FFlag gating                                                                                   |
| Inner shadow       | `shadow-inner`                                                                        | ❌ Not ported | ➖ Unlikely to be ported | `UIShadow` has no inset mode                                                                        |
| 📌 Spacing          | `m-*`, `p-*`, `gap-*`                                                                 | ❌ Not ported | ➕ Likely to be ported   | Will map to `::UIPadding` / `UIListLayout`                                                          |
| 📌 Sizing           | `w-*`, `h-*`, `min-*`, `max-*`                                                        | ❌ Not ported | ➕ Likely to be ported   | Will write `Size` / `UISizeConstraint`                                                              |
| Layout             | `flex`, `grid`, `block`, `hidden`, position utilities                                 | ❌ Not ported | ➕ Likely to be ported   | Maps to `UIListLayout` / `UIGridLayout` / `Visible` / `Position`                                    |
| 📌 Background color | `bg-{hue}-{shade}`                                                                    | ❌ Not ported | ➕ Likely to be ported   | Will write `BackgroundColor3`                                                                       |
| Z-index            | `z-{0,10,20,30,40,50}`                                                                | ✔️ Ported     |                         | Maps directly to `ZIndex`; `z-auto` skipped (no Roblox equivalent)                                  |
| Opacity            | `opacity-*`                                                                           | ❌ Not ported | 🔁 Revisit if needed     | Splits between `BackgroundTransparency` and `TextTransparency` — class semantics need a design call |
| Effects            | `blur-*`, `brightness-*`                                                              | ❌ Not ported | ➖ Unlikely to be ported | Roblox post-effects live on `Lighting`, not `GuiObject`                                             |

## Palette

The full Tailwind v3 color palette ships out of the box: `slate`, `gray`, `zinc`, `neutral`, `stone`, `red`, `orange`, `amber`, `yellow`, `lime`, `green`, `emerald`, `teal`, `cyan`, `sky`, `blue`, `indigo`, `violet`, `purple`, `fuchsia`, `pink`, `rose` — each with shades `50` → `950`. Every color utility (`text-*`, `border-*`, `ring-*`, `shadow-*`) reads from the same palette.
